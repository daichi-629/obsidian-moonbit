# Obsidian MoonBit

Reusable MoonBit integration for Obsidian plugin authors.

This repository now implements the package split described in
[`docs/moonbit-library-design.md`](/home/daichi/ghq/github.com/daichi-629/obsidian-plugin-moonbit-base/docs/moonbit-library-design.md):

- `@obsidian-moonbit/obsidian-moonbit`
- `@obsidian-moonbit/obsidian-moonbit-obsidian-api`
- `@obsidian-moonbit/obsidian-moonbit-esbuild-plugin`

## Install

Published package:

```bash
npm install obsidian-moonbit
```

Directly from GitHub without publishing:

```bash
npm install github:daichi-629/obsidian-moonbit
```

## Public API

Plugin authors use the same imports for both published and GitHub-installed builds:

```ts
import { loadMoonBit } from "obsidian-moonbit/obsidian-api";
import { moonBitEsbuildPlugin } from "obsidian-moonbit/esbuild-plugin";
import demoModule from "./moonbit/demo.mbt";

const wasm = await loadMoonBit(this, demoModule);
```

The imported `.mbt` module is an internal artifact object generated by the esbuild
plugin. Hashing, base64 embedding, cache paths, and binary persistence stay behind
`loadMoonBit(...)`.

## Workspace packages

`packages/obsidian-moonbit`
Core runtime for embedded MoonBit WebAssembly modules. This package is independent
from the Obsidian API.

`packages/obsidian-moonbit-obsidian-api`
Obsidian-specific adapter that accepts a `Plugin`, stores cached wasm files under
the plugin directory, and delegates instantiation to the core runtime.

`packages/obsidian-moonbit-esbuild-plugin`
Build-time integration for `.mbt` imports. It runs `moon build`, locates the wasm,
and emits the embedded module descriptor consumed by `loadMoonBit(...)`.

These workspace packages are implementation details. The supported consumer surface
is the root package `obsidian-moonbit` and its subpath exports.
